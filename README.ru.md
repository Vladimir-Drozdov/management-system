# Zoo Management System (Console Application)

Консольное backend-приложение для управления животными и инвентарём в зоопарке.

Проект сосредоточен на доменном моделировании, применении принципов SOLID и внедрении зависимостей
с использованием DI-контейнера Spring. Включает unit-тесты и чёткое разделение бизнес-логики.

---

## Функциональность

- Добавление животных с обязательной проверкой здоровья через ветеринарную клинику
- Управление инвентарём (животные также считаются объектами инвентаризации)
- Расчёт общего суточного потребления корма
- Формирование списка животных, допущенных в контактный зоопарк
- Консольное меню для взаимодействия с приложением

---

## Обзор архитектуры

Приложение построено вокруг чистой доменной модели с чётким разделением ответственности.

**Основные компоненты:**
- `Animal` и его наследники — доменные модели животных
- `IVetClinic` / `VetClinic` — абстракция проверки здоровья животных
- `IInventory` / `Thing` — абстракция инвентаря
- `Zoo` — основной доменный сервис
- `AppConfig` — конфигурация Spring DI
- `Main` — точка входа консольного приложения

---

## Принципы SOLID

- **Single Responsibility** — доменные сервисы и модели имеют чётко определённые обязанности
- **Open/Closed** — новые типы животных можно добавлять без изменения существующей логики
- **Liskov Substitution** — все наследники `Animal` взаимозаменяемы через базовый тип
- **Interface Segregation** — отдельные интерфейсы для «живых» сущностей и инвентаря
- **Dependency Inversion** — модули высокого уровня зависят от абстракций, а не от реализаций

---

## Внедрение зависимостей

Spring используется в качестве DI-контейнера для отделения создания объектов
от бизнес-логики. Зависимости настраиваются в `AppConfig` с использованием Java-конфигурации.

## Тестирование

Проект включает unit-тесты с покрытием более 60%.  
Тесты сфокусированы на основной бизнес-логике и доменном поведении.

---

## CI

В проекте используется GitHub Actions для непрерывной интеграции.

Каждый push и pull request запускает:
- сборку проекта
- выполнение unit-тестов

Используются Java 21 и Maven Wrapper.

---

## Как запустить

### Требования
- Java 21+

### Сборка и запуск
./mvnw clean package
./mvnw exec:java -Dexec.mainClass="org.example.app.Main"

### Запуск тестов
./mvnw test
